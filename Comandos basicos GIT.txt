**Configurações iniciais**
git config --global user.name "seu nome" --> O nome que você colocar aqui será o nome que ira aparecer toda vez que você commitar os arquivos;

git config --global user.email "seuemail@exemplo.com" --> coloque o email que você criou a conta do GitHub;

**Comandos basicos**
git init --> Cria um repositório Git nesta pasta e será gerenciado pelo git;

git status --> Verifica o status atual da pasta em que eu executei o git init (mostrando se o arquivo subiu ou não);

git add "nome-do-arquvo-que-quer-subir" --> Fazendo isso você esta preparando o arquivo para o controle de versionamento;

git add . --> Aqui você prepara mais de um arquivo para o controle de versionamento;
OBS sobre git add - Ele cria stage virtual dos arquivos que serão versionados, antes de subi-lo para o GitHub propriamente dito.

clear --> Limpa o prompt do git bash (parecido com cls do DOS);

git commit -m "aqui você vai informar o que foi feit no código/arquivo" --> é o comando para subir os arquivos;

git branch -M main --> Para garantir que o nome da pasta suba como main;

git remote add origin <endereço que irá informar assim que um novo repositório for criado> --> Atenção, pois la no GitHub preciso selecionar o botão SSH, pois os envios feitos pelo SSH são criptografados;

git push -u origin main --> Comando para subir os arquivos para o GitHub;



**Gerando SSH**
ssh-keygen -t ed25519 -C "your_email@example.com" /ou/ ssh-keygen -t rsa -b 4096 -C "your_email@example.com" --> Comando para criar a chave SSH
